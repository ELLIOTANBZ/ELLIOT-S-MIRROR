Build Daily CSV

Upload source files for audit, scorecard, ESS, interactions, training, performance banding, or project evidence. MIRROR will add the current org chart, profile, weights, and thresholds, then download one final CSV.

What Goes Into The File

The downloaded CSV includes admin configuration from this local MIRROR app:

  • Officer names, roles, teams, trained schemes, and reporting lines
  • Career profiles and target roles
  • Readiness weights
  • Readiness thresholds
  • Competency source weights

Source File Format

Uploaded source files must use MIRROR's expected column names so the app can recognise them.

  • Audit: officer_id, upload_date, total_score, competency columns
  • Scorecard: officer_id, upload_date, competency columns
  • ESS: officer_id, upload_date, rating, feedback, ESS Valid
  • Interactions: officer_id, upload_date, case_id, member_query, officer_response
  • Training/profile: officer_id, Title, Provider, Training Type, Status, Assigned Date, Completed Date, Description, Competency Gap, Start Date In Position, Trained Schemes
  • Performance: officer_id, Performance Period, Performance Banding, Performance Score
  • Projects: officer_id, Project Name, Project Leads, Requirements, What Was Done, Project Lead Comments, Project Score
  • Competency source weights: Source Competency, Source Audit Weight, Source Scorecard Weight, Source Interaction Weight, Source Project Weight

Full Daily CSV Columns

The final daily CSV can use these columns:

officer_id, Officer Name, Officer Role, Manager ID, Team Name, Trained Schemes, Current Role, Target Role, Start Date In Position, Expected Tenure Years, Key Responsibilities, Target Responsibilities, upload_date, total_score, Courtesy, Confidentiality, Comprehend Intent, Comply - Email Writing SOG, Correct Information, Complete Information, Clear and Easy, Meaningful Conversations, Cultivate Digital Awareness, Verified Mistake, Personal Development, Team Development, Stakeholder Development, rating, feedback, ESS Valid, case_id, member_query, officer_response, Title, Provider, Training Type, Status, Assigned Date, Completed Date, Description, Competency Gap, Performance Period, Performance Banding, Performance Score, Project Name, Project Leads, Requirements, What Was Done, Project Lead Comments, Readiness Role, Core Weight, Functional Weight, Correspondence Weight, Performance Banding Weight, Projects Weight, Experience Weight, Threshold Stage, Threshold Metric, Threshold Display Name, Threshold Minimum Value, Threshold Unit, Threshold Sequence, Source Competency, Source Audit Weight, Source Scorecard Weight, Source Interaction Weight, Source Project Weight

Threshold Rows

Threshold columns are normal CSV columns. Each threshold is one row, for example: Threshold Stage = Meeting Expectations, Threshold Metric = core, Threshold Minimum Value = 65.

Project Updates

Project leads can download project updates from the Projects page. Admin can upload those project update CSVs here together with the other source files, then download one combined daily CSV for everyone.

Import Daily MIRROR File

Upload one CSV/XLSX containing the daily officer data for this local MIRROR app.

The file can include profile, audit, scorecard, ESS, interaction, training, performance banding, and project evidence columns.

Scorecard format

Scorecard files must include an officer name or ID column, an upload date column, and scorecard code columns such as P1, P2, Q1, and B2.

One row must contain the criteria names with weights, for example RAMP/GRIT: 5% Online Feedback: 2% (10%). Officer rows below that should contain the score earned for each scorecard column.
